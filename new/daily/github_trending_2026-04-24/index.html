OpenAI 发布 GPT 5.5

OpenAI 发布 GPT 5.5

2026 年 4 月 23 日，OpenAI 正式发布 GPT-5.5，将其定位为应对“真实工作”挑战的智能模型。此次迭代的核心不在于单轮问答的提升，而在于模型在复杂任务中表现出的高自主性，包括编写代码、数据分析、跨工具协作及文档自动化生成等。

从技术指标看，GPT-5.5 实现了系统性优化：在保持与上一代模型同等响应延迟的前提下，通过更高效的令牌调度机制减少了任务完成所需的计算成本。官方公布的测试数据显示，该模型在多项权威基准测试中表现显著，例如在评估复杂命令行操作的 Terminal-Bench 2.0 中，准确率达到 82.7%；在考察综合职业知识工作的 GDPval 测试中，得分提升至 84.9%。

针对不同使用场景，GPT-5.5 细分为标准版与 Pro 版。目前，ChatGPT 和 Codex 的订阅用户（包括 Plus、Pro、Business、Enterprise 及部分教育计划）已可获得访问权限。针对 API 开发者，OpenAI 预告了即将上线的接入服务：标准版 GPT-5.5 定价为每百万输入 5 美元、输出 30 美元，而具备更高准确度的 Pro 版本则设定为每百万输入 30 美元、输出 180 美元，并支持高达 100 万令牌的上下文窗口。

OpenAI 表示，GPT-5.5 经历了其预备框架下的完整评估，重点防御高级网络安全与生物学风险。通过与近 200 家早期合作伙伴的联调，官方部署了更精细的安全分类器，并针对关键基础设施用户开放了“网络安全可信访问”渠道，以确保在赋能防御能力的同时，严格控制高风险滥用。

腾讯开源 Hy3 preview，混元重建后首个旗舰预览模型

腾讯开源 Hy3 preview，混元重建后首个旗舰预览模型

4月23日，腾讯混元发布并开源 Hy3 preview。官方将其定义为混元训练体系重建后的首个模型，也是目前“最强的一次”语言模型发布，目标场景明确指向 Agent 工作负载，重点提升复杂推理、指令遵循、长上下文、代码与智能体能力。模型权重已同步上线 Hugging Face、ModelScope 和 GitCode。

规格上，Hy3 preview 采用快慢思考融合的 MoE 架构，总参数 295B，激活参数 21B，另含 3.8B MTP 层参数，支持 256K 上下文，使用 192 个 experts、top-8 激活与 BF16 精度。部署侧支持 vLLM、SGLang 和 OpenAI 兼容接口，腾讯云文档还给出了交错式思考与工具调用流程，以及 reasoning_effort 的 high/low 控制方式。

代码与 Agent 是本次升级最明显的方向。模型卡给出的公开成绩包括 SWE-bench Verified 74.4、TerminalBench 2.0 54.4，并称其在 BrowseComp、WideSearch、ClawEval 和 WildClawBench 等搜索与工具调用评测中表现扎实。需要注意的是，这次虽然开源，但采用的是 Tencent Hy Community License，而不是 Apache 或 MIT 一类更宽松的许可。

众议院外交事务委员会推进了20项获得两党支持的法案，旨在收紧美国对中国的人工智能（AI）和半导体技术的出口管制。相关措施包括提高民事处罚、建立举报人奖励机制、将诉讼时效延长至10年，并在海外部署至少30名出口管制官员。《匹配法案》（Match Act）将要求识别受限的芯片设备和中国晶圆厂，并推动盟友采用类似的限制措施。《人工智能监察法案》（AI Overwatch Act）将禁止向中国销售英伟达公司（Nvidia Corp.）的高端芯片（包括 Blackwell 和 H200 处理器），并赋予国会对许可的监督权。《芯片安全法案》（Chip Security Act）规定了反规避规则以遏制走私。所有法案目前仍处于早期阶段，并可能被纳入年度国防法案中。

Qwen 上线 Qwen3.6-27B，27B 开源模型继续压向代码代理

Qwen 已上线 Qwen3.6-27B。按官方模型卡，它是一款带视觉编码器的 27B 参数模型，采用 Apache 2.0 许可证，兼容 Transformers、vLLM、SGLang 等主流推理框架，原生上下文长度为 262,144 tokens，并标注可扩展到 1,010,000 tokens

这代模型的主轴仍然是更贴近真实开发流程。官方将核心改进概括为 agentic coding 和 thinking preservation，前者覆盖前端工作流与仓库级推理，后者用于在多轮交互里保留历史推理上下文，以减少反复迭代时的额外开销。Qwen 官方 GitHub 仓库也把 Qwen3.6 的整体方向概括为 stability 和 real-world utility。

Qwen3.6-27B 相比 Qwen3.5-27B 在代码与代理任务上提升明显，例如 SWE-bench Verified 从 75.0 提升到 77.2，Terminal-Bench 2.0 从 41.6 提升到 59.3，SkillsBench Avg5 从 27.2 提升到 48.2；在综合推理和多模态任务上，它给出的分数包括 AIME 2026 的 94.1、MMMU 的 82.9，以及 AndroidWorld 的 70.3。整体看，这是一款继续把长上下文、工具使用和代码代理做实的 27B 档开源模型。
