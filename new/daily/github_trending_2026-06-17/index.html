Z.ai 已在 Hugging Face 发布 GLM-5.2 权重。官方集合目前包含 zai-org/GLM-5.2 与 zai-org/GLM-5.2-FP8 两个条目，分别对应基础版本和 FP8 版本；模型页显示许可证为 MIT，模型规模为 753B 参数。

Arch Linux 暂停 AUR 新用户注册以清理恶意软件包

Arch Linux 暂停 AUR 新用户注册以清理恶意软件包

由于遭遇大规模恶意软件包更新事件，Linux 社区发行版 Arch Linux 已暂停其用户软件仓库（AUR）的新用户注册服务。据悉，此次事件因攻击者滥用 AUR 的孤立软件包接管机制而起。社区最初报告受影响软件包约 400 个，但后续追踪显示受灾数量已超过 1500 个。

目前，AUR 平台整体保持在线，但官方已限制了新账号注册等可能扩大风险的操作。维护团队正全力追溯恶意提交并清理相关漏洞。官方提醒广大用户，在安装或更新 AUR 软件包（尤其是近期更换过维护者的工具）时，务必仔细检查 PKGBUILD 脚本及安装逻辑，以确保系统安全。

Anthropic 暂停 Claude Agent SDK 计费调整，程序化调用暂继续计入订阅额度

Anthropic 暂停 Claude Agent SDK 计费调整，程序化调用暂继续计入订阅额度

Anthropic 已暂停原定于 2026 年 6 月 15 日生效的 Claude Agent SDK 用量政策调整。根据官方帮助中心更新，Claude Agent SDK、claude -p 非交互模式以及基于 Agent SDK 的第三方应用，现阶段仍会从用户 Claude 订阅计划的使用额度中扣除。此前宣布的独立 Agent SDK 月度 credit 暂不开放。

这意味着 Anthropic 暂缓了将程序化调用从普通订阅额度中剥离的计划。原方案中，Claude Agent SDK、claude -p、Claude Code GitHub Actions 和第三方 Agent SDK 应用将转入单独的月度 credit 池，超出后再通过 usage credits 或标准 API 费率计费。

这一变化对开发者生态具有直接影响。对依赖 Claude Code、Agent SDK 和自动化工作流的用户而言，现有使用方式暂时不会被强制迁移到更高成本的独立计费路径。它也显示 Anthropic 正在重新评估 Claude 订阅、开发者工具和第三方应用之间的关系。

来源：Anthropic Help Center
