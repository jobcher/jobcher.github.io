Avalonia鸿蒙适配项目宣布停止更新

开发者布布近日宣布，OpenHarmony-NET组织下的Avalonia适配鸿蒙项目正式停止更新并归档。该决策源于适配工作的商业合作争议。据开发者透露，在Avalonia与华为洽谈合作期间，其适配工作转为闭源开发。然而，相关技术信息随后出现在国内供应商招标平台，最终由外包公司中标并接手后续开发工作。
开发者表示，由于外包公司已介入并可能在相关政策补贴下推进后续开源工作，原有开源项目不再具备公开更新的必要。目前，该组织名下的相关项目已全部停止维护。
开发者呼吁相关企业在引用已有代码时遵循MIT协议，并保留原作者的版权声明。

Claude 桌面客户端第三方模型兼容性问题获缓解方案

受近期版本更新影响，Claude 桌面客户端因加强了模型名称校验规则，导致部分通过 API 接入的第三方模型无法正常调用。技术人员发现，系统目前会对模型标识符进行格式审查，非原生命名格式可能触发校验失败。

针对该问题，社区已提出有效的避障方案。用户只需在原模型名称前添加 claude- 前缀（例如将 deepseek-v4-pro 修改为 claude-deepseek-v4-pro），即可绕过名称校验。经实测，该方法可使受影响的第三方模型在最新版桌面端恢复正常运行。
