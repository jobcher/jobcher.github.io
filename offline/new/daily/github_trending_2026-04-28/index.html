Claude Code 仍提供三种模型切换方式：在交互界面使用 /model 命令、启动时通过 --model 指定单次会话模型，或通过 ANTHROPIC_MODEL 环境变量设置默认模型。

GitHub Copilot 将转向按量计费

GitHub Copilot 将转向按量计费

GitHub 宣布，所有 GitHub Copilot 方案将从 2026 年 6 月 1 日起迁移至按量计费。原有的 premium request units 将被 GitHub AI Credits 取代，消耗将按输入、输出与缓存 token 计算，并依据各模型公开 API 费率扣减。

基础订阅价格不变。Copilot Pro 仍为 10 美元/月，Pro+ 为 39 美元/月，Business 为 19 美元/用户/月，Enterprise 为 39 美元/用户/月。代码补全与 Next Edit suggestions 仍包含在所有计划内，不消耗 AI Credits。

个人月付用户将在 6 月 1 日自动迁移。年付 Pro 与 Pro+ 用户可保留现有 premium request 计费直到订阅到期，但模型倍率将在 6 月 1 日上调；订阅到期后将转入 Copilot Free，并可选择升级为月付计划。

企业客户将获得过渡期额度。6 月至 8 月，Copilot Business 每用户每月包含 30 美元 AI Credits，Copilot Enterprise 每用户每月包含 70 美元 AI Credits。GitHub 还将支持组织内额度池化，并提供企业、成本中心和用户级预算控制。

来源：GitHub Blog

中国央行缩减MLF操作规模以回笼市场流动性

中国人民银行近期通过中期借贷便利（MLF）操作实现净回笼资金2000亿元，这是自2025年2月以来首次通过该工具回笼流动性。此举旨在应对银行间市场流动性过剩及贷款需求疲软导致的借贷成本偏低问题。通过减少中长期资金供应并维持短期资金稳定，央行正逐步优化流动性结构。
分析人士指出，此次资金回笼更多反映了当前市场对高成本政策资金的需求不足，而非政策的主动收紧。目前银行间拆借利率及存单利率均处于历史低位，甚至低于MLF利率。未来流动性回归常态的关键，仍取决于实体经济信贷需求的进一步修复。

来源：彭博社

QEMU 11.0 正式发布：增强虚拟化安全与硬件扩展支持

QEMU 11.0 正式发布：增强虚拟化安全与硬件扩展支持

开源虚拟化工具 QEMU 11.0 版本现已发布。新版本显著提升了安全性与兼容性，包括在 KVM 模式下新增对英特尔控制流强制技术（CET）虚拟化的支持，并支持 AMD SEV-SNP 与 Intel TDX 虚拟机的重置功能。此外，新版引入了原生 Nitro Enclaves 加速器以支持 AWS 环境，并增加了英特尔至强 Diamond Rapids 及 MIPS P8700 等处理器模型。

在架构优化方面，新版本更新了 RISC-V 与 LoongArch 的指令集扩展支持，同时清理了过时的 i440FX 和 Q35 机器模型。此次更新进一步强化了 QEMU 在 Linux 虚拟化技术栈中的核心地位，为跨平台模拟与云端安全提供了更完善的硬件仿真支持。
